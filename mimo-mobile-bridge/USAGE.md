# MiMo Mobile Bridge 使用方法

## 1. 项目概述

MiMo Mobile Bridge 是一个手机端远程控制 MiMoCode 的解决方案，包含两部分：

| 组件 | 目录 | 说明 |
|------|------|------|
| 电脑端插件 | `plugin/` | TypeScript，启动 WebSocket 服务器 |
| 手机端 App | `app/` | Flutter，连接电脑端进行远程对话 |

### 工作原理

```
手机 App ──WebSocket──> 电脑端插件 ──SDK──> MiMoCode
     ↑                      │
     └── 事件转发 <──────────┘
```

## 2. 环境要求

### 电脑端

| 项目 | 要求 |
|------|------|
| Node.js | >= 18.0.0 |
| MiMoCode | 已安装并运行 |
| 网络 | 手机和电脑在同一局域网 |

### 手机端

| 项目 | 要求 |
|------|------|
| Flutter SDK | >= 3.0.0 |
| Android | >= 6.0 (API 23) |
| iOS | >= 12.0 |

## 3. 电脑端插件安装

### 3.1 编译插件

```bash
cd d:\anmimocode\mimo-mobile-bridge\plugin
npm install
npm run build
```

编译后输出到 `plugin/dist/` 目录。

### 3.2 注册到 MiMoCode

在 MiMoCode 的配置文件中添加插件引用：

**方式一：项目级配置**

在项目根目录创建或编辑 `.mimocode/mimocode.json`：

```json
{
  "plugin": [
    "d:/anmimocode/mimo-mobile-bridge/plugin"
  ]
}
```

**方式二：全局配置**

编辑 `~/.mimocode/mimocode.json`：

```json
{
  "plugin": [
    "d:/anmimocode/mimo-mobile-bridge/plugin"
  ]
}
```

### 3.3 启动 MiMoCode

正常启动 MiMoCode，插件会自动加载并启动 WebSocket 服务器：

```
[MiMo Mobile Bridge] WebSocket 服务器已启动，监听 0.0.0.0:8765
```

### 3.4 获取电脑 IP 地址

手机需要连接到电脑的 IP：

```bash
# Windows
ipconfig

# 找到 "IPv4 地址"，例如 192.168.1.100
```

### 3.5 修改配对码（可选）

默认配对码为 `000000`，可在 `plugin/src/index.ts` 中修改：

```typescript
const DEFAULT_PAIRING_CODE = "你的配对码"
```

修改后重新编译：`npm run build`

## 5. 连接流程

### 5.1 首次连接

1. **启动电脑端 MiMoCode**（确保插件已加载）
2. **打开手机端 App**
3. App 自动进入连接设置界面：

```
┌─────────────────────────┐
│      连接设置            │
├─────────────────────────┤
│  服务器地址             │
│  ┌───────────────────┐  │
│  │ 192.168.1.100     │  │
│  └───────────────────┘  │
│                         │
│  端口                   │
│  ┌───────────────────┐  │
│  │ 8765              │  │
│  └───────────────────┘  │
│                         │
│  配对码                 │
│  ┌───────────────────┐  │
│  │ 000000            │  │
│  └───────────────────┘  │
│                         │
│      [连接]              │
└─────────────────────────┘
```

4. 输入电脑 IP、端口（默认 8765）、配对码（默认 000000）
5. 点击「连接」
6. 连接成功后自动进入项目列表

### 5.2 自动重连

- App 关闭后重新打开会自动尝试连接上次的服务器
- 连接断开时会自动重连（最多 5 次，间隔 3 秒）
- 配对码错误不会自动重连

### 6.4 胶囊通知（前台）

App 在前台时，顶部显示胶囊通知：

```
┌─────────────────────────┐
│  ⚡ 执行中  编译项目 65%  │ ← 收起状态
│  [仅此一次] [始终允许] [拒绝] │ ← 快捷按钮
└─────────────────────────┘
```

- **点击胶囊**：展开显示详情
- **快捷按钮**：可直接点击操作
- **左滑**：关闭通知
- **长按**：显示快速操作菜单

通知类型：

| 类型 | 说明 | 自动消失 |
|------|------|----------|
| 权限请求 | 需要用户确认操作 | 否（常驻） |
| 工具执行 | 工具执行进度和结果 | 完成后 2 秒 |
| 任务完成 | 任务结束通知 | 8 秒 |
| 错误警告 | 执行出错 | 15 秒 |

### 6.5 权限确认

当 AI 需要执行写文件、运行命令等操作时：

**前台 - 应用内弹窗：**

```
┌─────────────────────────┐
│      ⚠️ 请求写入权限     │
│                          │
│  AI 助手想要修改以下文件  │
│                          │
│  操作类型: 写入文件       │
│  目标文件: src/config.ts  │
│  所属项目: MiMo-Code      │
│  所属会话: 修复登录bug    │
│                          │
│  ┌────────────────────┐  │
│  │   ✓ 仅此一次        │  │ ← 蓝色（主要）
│  └────────────────────┘  │
│  ┌────────────────────┐  │
│  │   始终允许此操作    │  │ ← 绿色
│  └────────────────────┘  │
│  ┌────────────────────┐  │
│  │      拒绝           │  │ ← 红色
│  └────────────────────┘  │
└─────────────────────────┘
```

**后台 - 系统通知：**

```
┌─────────────────────────┐
│ MiMo Code · 修复登录bug  │
│ ⚠️ 请求写入文件权限       │
│ src/pages/login.tsx      │
│ [仅此一次] [始终允许] [拒绝] │
└─────────────────────────┘
```

- **仅此一次**：允许本次操作
- **始终允许**：允许该类型操作不再询问
- **拒绝**：拒绝本次操作

## 7. 系统通知（后台）

### 7.1 通知渠道

| 渠道 | 用途 | 声音 | 振动 |
|------|------|------|------|
| 权限请求 | 权限确认 | ✅ | ✅ |
| 任务通知 | 任务完成 | ✅ | ❌ |
| 错误警告 | 执行出错 | ✅ | ✅ |
| 执行进度 | 工具执行 | ❌ | ❌ |
| 会话状态 | 状态变化 | ✅ | ❌ |

### 7.2 各场景通知行为

| 场景 | 通知方式 |
|------|----------|
| App 在前台 | 应用内胶囊通知（可展开操作） |
| App 在后台 | 系统横幅通知（可直接操作） |
| 锁屏状态 | 锁屏通知卡片（可直接操作） |

### 7.3 通知操作

权限请求通知支持直接在通知上操作：
- 点击「仅此一次」→ 发送允许回复 + 关闭通知
- 点击「始终允许」→ 发送始终允许回复 + 关闭通知
- 点击「拒绝」→ 发送拒绝回复 + 关闭通知
- 点击通知本体 → 打开 App 进入对应会话

## 8. 配置参数

### 8.1 插件端配置

在 `plugin/src/index.ts` 中修改：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `port` | 8765 | WebSocket 端口 |
| `host` | 0.0.0.0 | 监听地址 |
| `pairingCode` | 000000 | 配对码 |
| `autoAllowRead` | true | 自动允许读取操作 |
| `timeout` | 30000 | 超时时间（ms） |

### 8.2 手机端配置

在 `app/lib/services/websocket_service.dart` 中修改：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `defaultPort` | 8765 | 默认端口 |
| `connectTimeout` | 10秒 | 连接超时 |
| `reconnectInterval` | 3秒 | 重连间隔 |
| `maxReconnectAttempts` | 5 | 最大重连次数 |

## 9. 常见问题

### Q: 连接失败怎么办？

1. 检查手机和电脑是否在同一局域网（同一 WiFi）
2. 检查电脑 IP 地址是否正确
3. 检查端口 8765 是否被占用
4. 检查防火墙是否阻止了连接
5. 确认 MiMoCode 已启动且插件已加载

### Q: 配对码错误怎么办？

确认手机端输入的配对码与电脑端 `plugin/src/index.ts` 中的 `DEFAULT_PAIRING_CODE` 一致。

### Q: 看不到项目列表？

1. 确认 MiMoCode 中已打开至少一个项目
2. 在 MiMoCode 中打开项目后再刷新手机端
3. 手机端无法创建新项目，只能查看电脑端已打开的项目

### Q: 消息发送后没有反应？

1. 检查 WebSocket 连接状态（应显示「已连接」）
2. 确认选择了正确的项目和会话
3. 查看 MiMoCode 是否正在处理（状态为「执行中」）
4. 检查电脑端 MiMoCode 的 AI Provider 是否配置正确

### Q: 权限弹窗一直不消失？

权限请求通知是常驻的，直到用户点击操作按钮。如果不需要该操作，点击「拒绝」即可关闭。

### Q: 后台收不到通知？

1. 检查手机系统设置，允许 MiMo Mobile 发送通知
2. Android: 设置 → 应用 → MiMo Mobile → 通知 → 全部允许
3. iOS: 设置 → MiMo Mobile → 通知 → 允许通知
4. 确认 App 在后台时 WebSocket 连接未断开
